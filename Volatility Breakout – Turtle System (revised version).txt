  Volatility Breakout – Turtle System style
  Entry: when price exceeds highest high or lowest low of last N bars
}

Inputs: 
    Length(20),    
    size(1);   
    
Variables:
    HighestHigh(0),
    LowestLow(0),
    LongEntry(0),
    ShortEntry(0),
    LongStop(0),
    ShortStop(0);

HighestHigh = Highest(High, Length)[1];   
LowestLow = Lowest(Low, Length)[1];       

LongEntry = HighestHigh;
ShortEntry = LowestLow;

{ Long entry breakout }
If MarketPosition = 0 Then Begin
    If Close > HighestHigh Then
        Buy size contract next bar at LongEntry stop;
End;

{ Short entry breakout }
If MarketPosition = 0 Then Begin
    If Close < LowestLow Then
        SellShort size contract next bar at ShortEntry stop;
End;

if marketposition <> 0 then begin
    sell size contract next bar at LowestLow stop;
    buytocover size contract next bar  HighestHigh stop;
end;

# Volatility Breakout – Turtle System (revised version)

**Source file:** `src/volatility_breakout_turtle.EL`

## Implemented Logic

- **Long Entry:** When the price closes above the highest high of the last `Length` bars (excluding the current bar), a `buy stop` order is placed at that high (`LongEntry`).
- **Short Entry:** When the price closes below the lowest low of the last `Length` bars, a `sell short stop` order is placed at that low (`ShortEntry`).
- **Exit (stop loss/trailing):** If in a position (long or short), two exit orders are placed:
  - For long positions: `sell` at `LowestLow` (the low of the lookback period)
  - For short positions: `buytocover` at `HighestHigh` (the high of the lookback period)
  - The order not relevant to the current direction is automatically ignored by the system.

## Why It's Interesting

- **Entry with close confirmation:** The signal is generated by the close beyond the breakout level, but the order is executed at the breakout price itself. This reduces false signals.
- **Symmetric (channel) exit:** The stop loss is based on the opposite channel level (LowestLow for longs, HighestHigh for shorts). Very simple, robust logic typical of trend following systems.
- **Unified stop management:** A single conditional block handles both directions, reducing code redundancy.
- **Designed for futures:** The use of `contract` (instead of `shares`) indicates the system is oriented to futures markets.
